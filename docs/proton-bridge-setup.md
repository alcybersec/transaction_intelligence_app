# Proton Mail Bridge Setup Guide

This guide covers setting up Proton Mail Bridge for headless Linux servers to enable email ingestion in the Transaction Intelligence App.

## Prerequisites

1. **Proton Mail Plus account or higher** (Bridge requires a paid subscription)
2. **Linux server** with systemd (tested on Ubuntu 22.04, Debian 12)
3. **X11 or headless support** (Bridge needs initial GUI login, then can run headless)

## Installation

### Download Bridge CLI

```bash
# Download the latest Bridge CLI from Proton's official releases
# Visit: https://proton.me/mail/bridge#download

# For Debian/Ubuntu:
wget https://proton.me/download/bridge/protonmail-bridge_X.X.X-1_amd64.deb
sudo dpkg -i protonmail-bridge_X.X.X-1_amd64.deb
sudo apt-get install -f  # Install any missing dependencies

# For RHEL/Fedora:
wget https://proton.me/download/bridge/protonmail-bridge-X.X.X-1.x86_64.rpm
sudo rpm -i protonmail-bridge-X.X.X-1.x86_64.rpm
```

### Install Dependencies

```bash
# Required dependencies
sudo apt-get install -y gnome-keyring pass

# For headless operation, you'll need a keyring backend
# Option 1: Use pass (recommended for headless)
gpg --gen-key  # Generate a GPG key if you don't have one
pass init <your-gpg-key-id>
```

## Initial Setup (Requires GUI)

The first login must be done with GUI access (can be done via VNC or SSH X11 forwarding).

### Option A: SSH X11 Forwarding

```bash
# From your local machine
ssh -X user@server

# On the server
protonmail-bridge --cli
```

### Option B: VNC/Desktop Access

```bash
# Start the Bridge GUI
protonmail-bridge
```

### Login Process

1. Launch Bridge
2. Click "Sign In" or run `login` in CLI mode
3. Enter your Proton Mail credentials
4. Complete 2FA if enabled
5. Bridge will generate IMAP/SMTP credentials

### Get IMAP Credentials

After login, retrieve your Bridge credentials:

```bash
# In Bridge CLI
protonmail-bridge --cli

# Run these commands:
> info
# Shows your IMAP server details and password
# Default: localhost:1143 (IMAP), localhost:1025 (SMTP)
```

**Important:** The Bridge password is NOT your Proton account password. It's a unique password generated by Bridge.

## Headless/Systemd Service Setup

### Create Systemd Service

```bash
sudo tee /etc/systemd/system/protonmail-bridge.service << 'EOF'
[Unit]
Description=Proton Mail Bridge
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=YOUR_USERNAME
Environment=DISPLAY=:0
Environment=DBUS_SESSION_BUS_ADDRESS=unix:path=/run/user/1000/bus
ExecStart=/usr/bin/protonmail-bridge --noninteractive
Restart=on-failure
RestartSec=10

# Security hardening
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=read-only
ReadWritePaths=/home/YOUR_USERNAME/.config/protonmail/bridge-v3
ReadWritePaths=/home/YOUR_USERNAME/.local/share/protonmail/bridge-v3
ReadWritePaths=/home/YOUR_USERNAME/.cache/protonmail/bridge-v3

[Install]
WantedBy=multi-user.target
EOF
```

Replace `YOUR_USERNAME` with your actual username and adjust the user ID in `DBUS_SESSION_BUS_ADDRESS` if needed.

### Enable and Start Service

```bash
# Reload systemd
sudo systemctl daemon-reload

# Enable on boot
sudo systemctl enable protonmail-bridge

# Start the service
sudo systemctl start protonmail-bridge

# Check status
sudo systemctl status protonmail-bridge

# View logs
journalctl -u protonmail-bridge -f
```

## Environment Configuration

Add these to your Transaction Intelligence `.env` file:

```bash
# Proton Mail Bridge IMAP
IMAP_HOST=127.0.0.1        # or host.docker.internal if running in Docker
IMAP_PORT=1143             # Bridge default IMAP port
IMAP_USER=your-proton-email@protonmail.com
IMAP_PASSWORD=your-bridge-generated-password  # NOT your Proton password!
```

### Docker Configuration

If running the app in Docker, Bridge runs on the host:

```bash
# In .env
IMAP_HOST=host.docker.internal  # Access host from Docker
IMAP_PORT=1143
```

Make sure Docker can resolve `host.docker.internal` (works by default on Docker Desktop, may need extra config on Linux).

## Verification

### Test IMAP Connection

```bash
# Install openssl if not present
sudo apt-get install openssl

# Test connection (STARTTLS)
openssl s_client -starttls imap -connect 127.0.0.1:1143

# You should see the IMAP server greeting
# Type: a LOGIN your-email@protonmail.com your-bridge-password
# Should respond: a OK Logged in
```

### Test from App

```bash
# Check if the worker can connect
cd /path/to/transaction_intelligence_app
docker compose logs worker | grep -i imap

# Or run a manual test
docker compose exec backend python -c "
from app.config import settings
print(f'IMAP Host: {settings.imap_host}')
print(f'IMAP Port: {settings.imap_port}')
print(f'IMAP User: {settings.imap_user}')
"
```

## Troubleshooting

### Connection Refused (ECONNREFUSED)

**Symptoms:** `Connection refused` or `Cannot connect to IMAP server`

**Solutions:**
1. Verify Bridge is running: `systemctl status protonmail-bridge`
2. Check Bridge is listening: `ss -tlnp | grep 1143`
3. Ensure firewall allows local connections: `sudo ufw allow from 127.0.0.1 to any port 1143`
4. If using Docker, ensure `host.docker.internal` resolves or use the host's actual IP

### Authentication Failed

**Symptoms:** `LOGIN failed` or `Authentication unsuccessful`

**Solutions:**
1. Use the Bridge-generated password, NOT your Proton password
2. Re-check credentials: `protonmail-bridge --cli` then `info`
3. Try regenerating the Bridge password
4. Ensure 2FA was completed during initial setup

### Bridge Crashes on Startup

**Symptoms:** Service fails to start, exits immediately

**Solutions:**
1. Check logs: `journalctl -u protonmail-bridge -e`
2. Ensure keyring is accessible: `pass show`
3. Try running manually first: `protonmail-bridge --cli`
4. Check disk space: `df -h`

### Emails Not Being Fetched

**Symptoms:** Bridge works but no emails appear

**Solutions:**
1. Check the INBOX folder exists and has emails
2. Verify email filters in Proton Mail web
3. Check worker logs for parsing errors
4. Ensure the sender patterns match your bank's email address

### "Keyring not available" Error

**Symptoms:** `secret-service: cannot open session`

**Solutions:**
```bash
# Initialize pass keyring
gpg --gen-key
pass init <your-gpg-key-id>

# Or start gnome-keyring manually
eval $(gnome-keyring-daemon --start)
export SSH_AUTH_SOCK
```

## Security Notes

1. **Bridge password:** Store the Bridge-generated password securely. It's different from your Proton account password.

2. **Local only:** Bridge only listens on localhost by default. Do not expose ports 1143/1025 to the network.

3. **Encryption:** Bridge maintains end-to-end encryption. Emails are decrypted locally only.

4. **Systemd hardening:** The service file includes security options to restrict Bridge's access.

## Maintenance

### Update Bridge

```bash
# Stop the service
sudo systemctl stop protonmail-bridge

# Install the new version
sudo dpkg -i protonmail-bridge_NEW_VERSION_amd64.deb

# Restart
sudo systemctl start protonmail-bridge
```

### Refresh Login

If Bridge session expires:

```bash
# Stop service
sudo systemctl stop protonmail-bridge

# Re-login via CLI
protonmail-bridge --cli
> login

# Restart service
sudo systemctl start protonmail-bridge
```

### Check Bridge Version

```bash
protonmail-bridge --version
```
